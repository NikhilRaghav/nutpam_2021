EVENT CAPTION CODESENSE

Syntax Error : expected

A big shout out to the curious programmers out there, want to test your coding and logical thinking skills? The wait is over, do participate in 'CODE SENSE' a Coding + MCQ event and be ready to test your skills!!


Description :

  • The event is conducted as a single round and it's a solo event
  • The competition will be conducted in "Skillenza" platform .
  • The competition will be a Coding + MCQ round .
  • Language Support: Python 2, Python 3, Ruby, JavaScript, Scala, Go, C, PERL, Java 8, Java 6, C++, C#, Kotlin, SQL Server, R, Haskell

Rules & Regulations:

  • Plagiarism and cheating by any means will not be encouraged and leads to immediate disqualification
  • The score of an individual will be calculated by the code efficiency and solving duration.
  • Judge decision is the final and no other changes will be encouraged.
  • Posting solutions/test cases in discussion forum will lead to disqualification.
  • Leaderboard is calculated by atleast code submission time and score of the particular individual.

EVENT CAPTION DEBUG ENTITY

The wages of sin is debugging!

Event description :

  • Debug entity is an event which has three rounds.
  • Easy - 3 questions for 10 mins
  • Intermediate - 2 questions for 10 mins
  • Difficult - 1 questions for 10 mins
  • In this event participants will have to debug either in C or Python.
  • Participants have to be careful while answering questions and have to complete within the given stipulated time.
  • Individual participation.
  • Cash prize is awarded for the winning participant.

Rules & Regulations :

  • Participants who have registered is only eligible for this event.
  • Every participants should have installed discord on their system.
  • Link will be provided and the participants should join the event on time.
  • Event will be conducted in SkillEnza coding platform.
  • Before staring the event participants should choose their programming language (C/Python).
  • In case of any malpractice, participants will be disqualified immediately.

Time Duration :

Total time duration is 30 mins.

Team size :

1

EVENT CAPTION INNOVATION HUB

"Innovation is not only doing different things but also doing things differently and productively."


Innovation is more than having new ideas, it includes the process of successfully introducing them or making things happen in a new way. So turn your ideas into useful, practicable and commercial products or services.


DESCRIPTION :


  • Innovation Hub is a totally theme-based presentatio.
  • It is a technical event where the participants is required to submit the ideas.
  • Participants can use any form of media to present their idea like power points, live demonstrations (all on Google Meet).
  • Matching the theme (SDG) border, the best team will be grabbing the prize!
  • Participants will be judged based on spontaneity, knowledge about the topic, body language, communication skills and clarity.

Theme :

Sustainable Development Goals


RULES & REGULATIONS :

  • The ideas will then be put through an initial test to make sure there is no plagiarism.
  • Any idea found to be plagiarised will be immediately disqualified.
  • After the preliminary scanning the selected ideas will be announced after which the participants will be required to present their work to a pannel of esteemed judges.
  • Each team should present within 10 minutes of time.
  • The idea should match the Sustainable Development Goals.
  • Results will be purely based on the final judgement of the pannel members.
  • A team should submit one idea.
  • While presenting the idea, Team members must turn on the videos.

Team size :

Two

EVENT CAPTION INTERFACE ROYALE

"Design is Intelligence made Visible".

Are you fond of designing? Then Interface Royale is just for you. It is a Web Designing event where participants are required to design a website adhering to one of the 5 themes (which will be revealed during the event) using the Figma platform within the stipulated time.


Tool :

Figma

Duration :

2 hrs

Theme :

To be disclosed at the time of the event.

Platform :

Discord


RULES & REGULATIONS :

  • Each participant should have a Discord
  • Participants should join the event's (Interface Royale) discord server.
  • The designs should be created using Figma and hence participants should have a figma account beforehand .
  • Depending on the number of the teams registered, seperate rooms/channels will be allotted for the team.
  • The teams should be present inside the allotted room during the entire event.
  • Any one member of the team must present their screen during the course of designing.
  • If the presenting member of the team faces any issues, it should be intimated to the event coordinator and other member of the team should continue presenting their screen.
  • Themes for designing the webpage will be disclosed 10 mins before the start of the event.
  • No need to use any of the coding languages/scripts like HTML, CSS, JS.
  • The participants are required to design only the UI and need not implemented the same.

Team size :

Maximum 3

EVENT CAPTION SCHEME WITH SCHEMA

  • Participants should be careful while answering questions and have to finish within time no extra will be provided.
  • Individual participation.
  • Cash prize is awarded for the wining participant.

Rules & Regulations :

  • Participants who have registered is only eligible for this event.
  • Every participants should have installed discord on their system.
  • Link will be provided and the participants should join the event on time.
  • Event will be conducted in HackerRank coding platform.
  • This event consists of two rounds
    1. Multiple choice questions on DBMS (30 questions for 30 minutes).
    2. SQL queries (15 questions for 30 minutes).
  • In case of any malpractice, participants will be disqualified immediately.

Time Duration :

1 Hour

Team size :

One

EVENT CAPTION DEFY THE LIMITS

Defy the limits is an Gaming contest

Rules & Regulations :

  • Toxic behaviour is not entertained and may lead to direct elimination.
  • Substitute players is not mandatory.
  • Tactical timeouts will be available for each side once per match.

Qualifiers :

  • Every match will be an elimination round of best of 1.
  • Coin tosses will be performed to decide to choose the map and sides before each match.
  • Each team can ban a map before the starting of the match.
  • Facture map is out by default in the map pool.

Finals :

  • Finals will be a best of 3.
  • Each team gets to ban and pick one map each and the last map by coin toss.
